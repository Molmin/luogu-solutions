容斥原理。

最大值就是 $\text{min(a,b)}$，这个大家稍微思考一下就能知道。

最小值分两种情况：

1. $x+y \le n$：这种情况我们完全可以将回答 $x$ 的人的集合和 $y$ 的集合分离开来，让他们没有交集。
2. $x+y > n$：这种根据容斥原理，最小值是 $x+y-n$，可以去搜一下“叠木板问题”，小奥的，很简单的。

~~因为 6872 今天是来写最短代码的~~，所以我们要将最小值做优化。

我们知道，如果 $x+y \le n$，可以推出 $x+y-n \le 0$，而情况二的 $x+y-n>0$，所以我们可以输出 $\text{max(0, x+y-n)}$，来“卡”掉情况一。

下面是代码：

```cpp
#include <bits/stdc++.h>
int n, x, y;
int main(){
    std::cin >> n >> x >> y;
    std::cout << std::min(x, y) << " " << std::max(0, x + y - n);
}
```
