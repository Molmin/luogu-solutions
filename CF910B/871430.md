#### [题 目 传 送 门](https://www.luogu.com.cn/problem/CF910B)
#### [题意解析]

题目给的意思十分清楚，有无数根长度为 $n$ 的木条，可以切割木条但不能拼接木条，现在需要得到 $4$ 根长度为 $ a $ 的木条以及 $2$ 根长度为 $b$ 的木条，问最少需要几根完整的木条。

#### [思路分析]

题解中大佬们的做法震惊了本蒟.....全是暴力全排列或者 DFS，本蒟承担不起这些算法，一打就错，打代码 $10$ 分钟，查错 $60$ 分钟，所以在这里介绍一种更加浅显易懂，码量更小的做法，**数学计算**。

对于每一个数据，我们知道，只要找出 $2$ 根长度为 $b$ 的木条就行了，也就是说，对于长度为 $b$ 的木条来说，切割出的两根木条可能是从同一根木棍上切出，也可能是不同的两根木棍中切出。我们只要分别考虑这两种情况，算出最少需要的木棍就好了。

#### [贴上码码]

```cpp
#include<bits/stdc++.h>
using namespace std;
int n,a,b,mi=INT_MAX;
int main(){
	cin>>n>>b>>a;
	if((4-(n-a-a)/b)<=0)mi=min(mi,1);
	mi=min(mi,1+int(ceil(4-(n-a-a)/b/(n/b))));
	if((4-(n-a)/b*2)<=0)mi=min(mi,2);
	mi=min(mi,2+int(ceil((4-(n-a)/b*2)/(n/b))));
	cout<<mi;
	return 0;
}
```
