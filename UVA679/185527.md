可以看到，这道题如果暴力模拟不吸氧，就会愉快地TLE，所以我们要做一些优化。

想了好久突然发现，是不是**小球下落到的节点之间有什么联系！**

于是思考若干分钟，把代码写了出来：(因为实在不难，所以讲解都放在注释里）

```cpp
#include<iostream>
using namespace std;
int a,b,k,j;  //定义
int main(){
    int n;
    cin>>n;  //输入数据组数
    for(int i=0;i<n;i++){  //循环输入
        cin>>a>>b;  //对应题目中的d和i
        j=1;
        k=1;  //每次处理数据都要初始化
        while(k<a){  //如果还没有落到底就进行操作
            if(b%2==0){  //如果是偶数点(true)
                j=j*2+1;  //j标记向右
            }else{  //如果是奇数点(false)
                j*=2;  //j标记向左
            }
            k++;  //计数器要++
            b=(b+1)/2;  //实际下落
        }
        cout<<j<<endl;  //落到底就输出，注意格式
    }
    return 0;  //结束啦！
}
```

然后就70msAC啦。

注意本题b和j的作用要区分开，一个是实际下落，一个是记录下落。
