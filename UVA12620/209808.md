### 解题思路

我们都知道斐波那契数列在模的意义下是有循环节的

而此题模数小，只有 $100$，故循环节较短，所以我们考虑找到循环节

那么循环节是多少呢？

找循环节的算法大致是

把 $n$ 质因数分解,分解为 $p_1^{k_1}*p_2^{k_2}*...p_n^{k_n}$

分别计算斐波那契数列中在模 $p^m$ 的循环节长度，令长度分别为 $x_1,x_2,x_3.....x_n$

则循环节长度为 $lcm(x_1,x_2,x_3,....,x_n)$

那如何求斐波那契数列在模 $p^m$ 的循环节长度呢？

有定理：

斐波那契数列模 $p^m$ 的循环节长度为 $G(p)*p^{m-1}$ 

其中 $G(p)$ 为斐波那契数列在模 $p$ 意义下的循环节长度

对于 $p$ 小的，我们有 $G(2)=3,G(3)=8,G(5)=20$ 这些对于做这道题来说足够了（如果想了解更多请自行 $bdfs$）

因为 $100=2^2*5^2$，所以我们需要知道 $G(2)$ 和 $G(5)$

然后 $x_1=G(2)*2=6,x_2=G(5)*5=100$

$lcm(x1,x2)=lcm(6,100)=300$ 所以斐波那契数列在模 $100$ 的意义下循环节为 $300$

我们预处理出前 $300$ 个斐波那契数列

然后用前缀和数组记录

然后算就好了

具体的看 $AC$ 代码

不开 $O_2$ 用时 $0ms$ 成功抢到最优解

### AC代码
```cpp
#include<bits/stdc++.h>
#define ll long long
using namespace std;
int t;
ll n,m;
int a[301],sum[301];
int main(){
	cin>>t;
	a[1]=a[2]=1;sum[1]=1,sum[2]=2;
	for(int i=3;i<=300;i++) a[i]=(a[i-1]+a[i-2])%100,sum[i]=sum[i-1]+a[i];
	while(t--){
		scanf("%lld%lld",&n,&m);
		ll x,y;
		x=sum[300]*((n-1)/300)+sum[(n-1)%300];
		y=sum[300]*(m/300)+sum[m%300];
		printf("%lld\n",y-x);
	}
}
```
