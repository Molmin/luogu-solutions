首先我们需要明确经度的概念。样例一中的 `E` 代表东经，样例二中 `W` 代表西经，就是 $\texttt{East}$ 和 $\texttt{West}$ 的缩写。

为什么没有南北？因为南北是纬度。

在地理中，比如东经 $75$ 度，要写成：$75^\circ \text{E}$。

由此我们可以推断出：**经度指示南北，是纵向的**。这里和经度的东西是相反的。

接下来来介绍一下本文中粗体字中的“国际日期变更线”：

> 全球各地都以自己所看到的太阳的位置作为确定一天的标准，把与自己所在地方相应的地球另一面的一条经线作为“日期变更线”也叫国际日界线，这样就有许多条“日期变更线”，使用起来很不方便。为了解决这个问题，应该规定一条全世界共同的、可供对照的“日期变更线”。随着标准时计时的区时系统的确立，东西十二区重叠，计时相同但日期不同，为避免混乱，公认 $180^\circ$ 经线作为日期变更线，因为是以“格林尼治时间”为标准的日期变更线。这条“日期变更线”就叫“国际日期变更线”。

好的，下面我们来看题吧。题意很简单，求两个经度位置的时差。

地球上经度每隔 $15^\circ$ 时差就会差一小时。所以我们可以直接求出两个经度的差，去除以 $15$。但是从一个地方到另一个地方有两条路线（因为地球是圆形的），如果一条经过国际日期变更线，就要从另一边绕去求时差。什么情况会经过这条该死的线捏？相信你通过上边这段文字发现了，当一个在西经一个在东经的情况下，就要从另一边去求时差。

设一个经度为 $a$，另一个为 $b$，那么这种情况的时差就是：

$$
\frac{a-(-b)}{15}
=\frac{a+b}{15}
$$

接下来就是代码时间：

```cpp
#include <bits/stdc++.h>
using namespace std;

int main(){
    char c, C;
    int k, K;
    cin>>c>>k>>C>>K;
    if (c!=C) cout<<(k+K)/15<<endl; // 同经
    else cout<<abs(k-K)/15<<endl;
    return 0;
}
```
