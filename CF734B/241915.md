这道题的其他题解里把题意讲的很明白了，这篇题解里就不再讲了，主要讲一下贪心思路和证明方法。

要组成 256 需要 1 个 2, 1 个 5 和 1 个 6。

要组成 32 需要 1 个 2 和 1 个 3。

可以明显看出 5 和 6 只能用来组成 256， 所以我们可以把 $k5$ 和 $k6$ 合并成一个数，即 $min(k5,k6)$，因为最终可组成的 256 数量为 $min(k2,k5,k6)$，所以如此合并之后最终答案不会受到影响。为了讲述的方便，这里暂称它为 $k$。

化简后的题意就变成了：

要组成 256 需要 1 个 $k2$ 和 1 个 $k$。

要组成 32 需要 1 个 $k2$ 和 1 个 $k3$。

明显可以看出，尽可能多的组成 256 比 32 划算，因为组成这两个数字的消耗都是 2。

于是我们就可以贪心出一个 $O(1)$ 柿子：

$min(min(k5,k6),k2) * 256 + min(k3,k2 - min(min(k5,k6),k2)) * 32$

当然你也可以用三目运算符写：

$min(k5,k6) >= k2 ? k2 * 256 : min(k5,k6) * 256 + min(k3,k2 - min(k5,k6)) * 32$

为不知道三目运算符是什么的人介绍一下：

一个最简单的 if 语句：

```cpp
if (a > 3)
{
	a -= 2;
}
else 
	{
		a += 2;
	}
```

用三目运算符写就是：

```cpp
a > 3 ? a -= 2 : a += 2;
```

`?` 类似于如果（if），`:` 类似于否则（else）。

不过个人感觉三目运算符唯一不太好的地方就是如果不写 `:` 会报错。

比如说：

```cpp
a > 3 ? a -= 2;
```

这种写法就是错的，会CE。
