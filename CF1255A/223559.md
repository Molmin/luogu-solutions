题目分析：

考虑贪心。

首先考虑向上调的情况，如果差距过大，显然优先按 $5$，尽快向上调，这样的操作直到差值小于 $5$ 时停止。此时根据剩下的差值决定调整方案。具体的： $3$ 和 $4$ 需要再调两次，$1$ 和 $2$ 也要再调一次。

向下调思路也是一样的，先不断向下调整 $5$，然后根据剩下的决策。

------------
但其实还有一种可能情况：即，先调到一个更高的音量然后再下调或是先调到一个更低的音量然后再上调。

实际上，手推一下仅有的几种情况就会发现其实并不会更优，所以题目中的向下调不能越过 $0$，也并不会对结果造成任何影响。

------------
代码：

```cpp
#include<cstdio>
#include<algorithm>
using namespace std;
int ans,a,b,T;
int main(){
	scanf("%d",&T);
	while(T--){
		scanf("%d%d",&a,&b);
		if(a>b)swap(a,b);
		ans=(b-a)/5;
		if((b-a)%5==4||(b-a)%5==3)ans+=2;
		if((b-a)%5==2||(b-a)%5==1)ans++;
		printf("%d\n",ans);
	}
	return 0;
}
```
