这道题数据范围可达到$10^{100}$，因此$unsigned$ $long$ $long$装不下装不下装不下，$float$装不下装不下装不下！~~（重要的事情说三遍）~~

因而，我们把目光放到了更高精度和范围的浮点数上面。

|数据类型|关键字|大小|范围|有效数字|是否可以用于该题
|-|-|-|-|-|-|
|单精度|$float$|$4$字节|$[±3.4\times 10^{-38},±3.4\times 10^{38}]$|$7$位|否|
|双精度|$double$|$8$字节|$[±1.7\times 10^{-308},±1.7\times 10^{308}]$|$16$位|是|

本蒟蒻查找资料之后发现，在$C++$的某些编译器中，$long$ $double$与$double$的精度是相同的。而在$C$语言中，$long$ $double$比$double$的精度高出四位，范围也大一些。在这道题里面精度是没有多大影响的，所以不需要考虑太多。总之，这两个类型的区别与实际编译器有关。保险起见，请大家使用$long$ $double$吧（注意用$\%Lf$标识符）！

$C++ AC$代码：

```cpp
#include<cstdio>
#include<cmath>
long double n,p;
int main()
{
    while(scanf("%Lf%Lf",&n,&p)!=EOF)printf("%.0Lf\n",pow(p,1/n));
}
```
