题意：给你 $t$ 个字符串，每个字符只能是 $E$ 或 $N$，在第 $i$ 个位置为 $E(Equal)$ 就表示 $a_i=a_{i+1}$，为 $N(Not$ $equal)$ 就表示 $a_i\neq a_{i+1}$。特别的，当 $i$ = $n$ （即最后一个数）时，$a_{i+1}$ 变为 $a_1$。

求是否存在数组 $a$。

这道题因为不需要构造，且范围够大，所以说是不需要去一个一个字符尝试的。显然，这道题在绝大多数情况都是可以的。可以发现，当输出 no 时一定是最后一个 $a_{n}$ 与 $a_1$ 的关系产生矛盾。

这里举个例子：NEE 输出 no

这里不能构造的原因是 $a_1\neq a_2,a_2=a_3,a_3=a_1$

就是最后一个数与第一个数的关系矛盾，即 $a_1 \neq a_x$,$a_n=a_x,a_1=a_n$(这个矛盾）

上面的两个符号换过来也行。

那为什么会出现这种情况？多试试就能知道，当只出现一个 $N$ 时，就会出问题。

形象的解释一下：

设这个数组 $a$ 长度为 $n$，只有下标为 $x$ 的数对应的字符为 $N$，其余的都是 $E$。

![](https://cdn.luogu.com.cn/upload/image_hosting/8tnbfpwi.png)

在上图，根据数据应该有 $a_n=a_1$,但 $a_1\neq a_{x+1}$，所以矛盾。

因此，根据上图的规律，当这个字符串只有一个 $N$ 时矛盾，其余可以。

### AC代码：

```cpp
#include<bits/stdc++.h>
using namespace std;
int main(){
	int t;
	cin >> t;//t组数据 
	while(t--){
		int cnt = 0;//计数器 
		string str;
		cin >> str;//输入字符串 
		for(int i = 0; i < str.size(); i++){
			if(str[i] == 'N'){
				cnt++;//统计N的个数 
			}
		}
		if(cnt == 1){//只有1个，矛盾 
			puts("No");//输出no 
		}
		else{//其余情况成立 
			puts("Yes");//输出yes 
		}
	}
	return !0;
}
```
