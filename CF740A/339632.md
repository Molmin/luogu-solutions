先来分析一下题目。显然我们需要从$n$模$4$的余数下手，一个一个来看。

如果余$0$，显然不用买任何书，直接输出$0$即可。

如果余$1$，注意此时是差$3$本，而不是差$1$本。

这个时候有$3$中买书的方法：

第一种是买$3$套$1$本；

第二种是买$1$套$1$本，$1$套$2$本；

最后一种是买$1$套$3$本。

这里把三种方法分别简记为$3+0+0,1+1+0,0+0+1$。

计算总钱数的代码就很好写了，在完整代码中见吧。

继续，如果余$2$，有$2+0+0,0+1+0,0+0+2$三种方法。

值得一提的是$0+0+2$，虽然它多买了4本书，但是并不代表它一定更贵。所以这种情况还是要算的。

最后，如果余$3$，那么差$1$本。有：$1+0+0,0+1+1,0+0+3$三种方法。

思路就讲到这里，看代码！

```python
n,a,b,c=map(int,input().split()) #输入4个数
m=n%4 #计算余数
if m==0: print(0) #已经整除，直接输出0
if m==1: print(min(a*3,a+b,c)) #差3本
if m==2: print(min(a*2,b,c*2)) #差2本
if m==3: print(min(a,b+c,c*3)) #差1本
```
### 最后，一些FAQs：（大佬们请自动忽略）

Q1：为什么每一种情况都只有$3$种情况，不能更多了吗？

比如，$m=3$中，$1+2+0$不也是可以的吗？为什么不需要算？

A:但是这样的话，它的总钱数一定比$1+0+0$大，因为它多买了$2$套$2$本的数。其他情况也是如此，因此无需讨论。

Q2：求$a,b,c$的最小值，不需要写

```python
min(a,min(b,c))
```
吗？

A：**不需要**。PY中的min函数是无论多少项都可以计算的。
