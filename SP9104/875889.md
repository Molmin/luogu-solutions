# 分析
为了解决这个问题，我们可以使用组合数学的方法来计算不匹配的总方式。

首先，我们需要知道匹配问题的有效解决方案的数量。对于每个测试用例，有效解决方案的数量为 $N!$，即第一列的 $N$ 个单词在第二列中的 $N$ 个位置中的所有可能排列的数量。

然而，在这个问题中，我们要找到没有正确匹配的情况。因此，我们需要计算有多少种方式让第一列的单词全部错误地与第二列中的单词匹配。

考虑第一列中的单词，它们可以与第二列中的 $N-1$ 个单词匹配，因为它们不能与自己匹配。所以，第一个单词有 $N-1$ 种不匹配的选择。

接下来，考虑第二个单词，它也可以与第二列中的 $N-1$ 个单词匹配，但它不能与第一个单词匹配。所以，第二个单词有 $N-2$ 种不匹配的选择。

依此类推，第 $i$ 个单词有 $N-i$ 种不匹配的选择。

因此，所有单词不匹配的总方式数量为 $(N-1)\times(N-2)\times ... \times1=(N-1)!$。

所以，我们的解决方案是将有效解决方案的数量 $N!$ 减去所有单词不匹配的总方式数量 $(N-1)!$，即答案为 $N! - (N-1)!$。

对于每个测试用例，我们可以按照上述步骤计算出解决方案的总方式，然后将结果输出。

# 代码
```cpp
#include<bits/stdc++.h>
using namespace std;
int T,N;
int main(){
    cin>>T;
    while(T--){
        cin>>N;
        long long ans=1;
        for(int i=1;i<=N;i++)
            ans*=i;
        cout<<ans-ans/N<<'\n';
    }
    return 0;
}
```
