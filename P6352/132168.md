这道题目，我们可以用暴力枚举解决。

因为输入的三个数都在[-100,100]，所以，第四个数，估摸着不会超过[-300,300]。

暴力枚举第四个数，每次将这四个数排个序，看看两两之间差是否相同，如果可以，就输出答案，否则继续找。

那就没什么好说的了，毕竟它是一道红题嘛！

上代码啊：
```cpp
#include<bits/stdc++.h>
using namespace std;
int a[11];
int b[11];
int main()
{
	register int i;
	cin>>a[1]>>a[2]>>a[3];
	for(i=-300;i<=300;i++)
	{
		b[1]=a[1];
		b[2]=a[2];
		b[3]=a[3];
		b[4]=i;
		sort(b+1,b+5);
		if(b[2]-b[1]==b[3]-b[2]&&b[4]-b[3]==b[2]-b[1])
		{
			cout<<i<<endl;
			return 0;
		}
	}
	return 0;
}
```
