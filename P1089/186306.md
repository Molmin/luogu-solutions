import java.util.Scanner;

public class Main {
	public static void main(String args[]) {
		int b, i, c = 0;
		int sum = 300;
		int x = 0;
		int m = 0;
		double ssum = 0;
		Scanner input = new Scanner(System.in);
		for (i = 1; i <= 12; i++) {
			int a = input.nextInt();
			if (sum < a && x == 0) {
				x = i;
			} else {
				b = sum - a;
				if (b >= 100) {
					c = b % 100;
					m = m + b / 100; // m=1,
					sum = c + 300;
					ssum = c + m * 100 + m * 100 * 0.2;
				} else {
					sum = b + 300;
					ssum = b + m * 100 + m * 100 * 0.2;
				}
			}
		}
		if (x != 0) {
			System.out.println("-" + x);
		} else {
			System.out.println((int) ssum);
		}

	}
}

# 这道题的难点，主要在于逻辑上容易忽略一些东西，之前一直90分，第六个测试点一直AC不了，现在终于满分通过，现给大家分享一下
首先a是我们键盘输入的每个月预算的钱；sum是每个月我们手里的钱，第一个月我们手里肯定只有300，所以sum的初始值是300；那么b=sum-a 就是我们每个月还能剩余的钱。

接下来进行判断，判断我们剩余的钱是否大于100元，如果不大于100元，那么我们只需要将剩余的钱加上下个月妈妈给我们的300元，就是我们手里的钱了；如果我们剩余的钱大于或者等于100（容易忽略的点之一：一定要是大于等于100，不能只是大于100，因为等于100我们也要存到妈妈手里）那么我们就要将整百的钱存到妈妈手里，所以我们剩余的钱就只有c=b%100，这里的c就是我们剩余的钱；这里还要注意的是:我们的ssum（ssum代表最后我们手中会有多少钱）在两种情况下的结果是不一样的，当我们要存钱时，我们的ssum=c+ m * 100 + m * 100 * 0.2，因为要存正百的到妈妈手里，所以用c+；当我们的钱不够整百，不需要存钱的时候，我们的ssum= b + m * 100 + m * 100 * 0.2,因为我们的钱不够整百，所以用b+。

另外m代表我们存到妈妈手里的钱是几个100；i代表月份

还值得我们注意的是，我们最后输出的-X和ssum一定要在for循环之后输出，要不然就会你输入一个预算，它就显示一个ssum或者显示-X，这很明显不符合我们题目中的意思，我们题目中的意思是你输入完12个预算整数之后，再输出一个整数代表我们最后年末手里的钱或者第几个月出现透支。

最后说明一下之前第六个点一直AC不了的原因，因为 假如我们第二个月和第三个月的预算都大于我们手里的钱了，都透支了，但是因为我们是输入完12个月份之后，再输出第几个月透支，这时候应该输出的是-2，但是由于我们还在输入，i还在++，所以我们实际输出的是-3，这就不符合题意，所以我们在if (sum < a && x == 0)判断这里再加上一个询问x是否等于0，如果不等于0，那么我们就锁定了之前的X，而让i继续++去吧，这样就能防止有两个月都超出预算而最后的出的结果是后一个月的情况发生了！
