## Part1 Consideration

题目回顾：

张三（化名）有 $3$ 个整数 $a$，$b$，$c$。你需要选择一个整数 $m$，使 $a$，$b$，$c$ 中的任意一个数乘上它，并让之后的 $a$，$b$，$c$ 构成算术级数。

算术级数：一个数列中每一组相邻的两个数的差相等（即为等差数列），如：$[5,10,15]$，$[3,2,1]$，$[1,1,1]$，$[13,10,7]$。而 $[1,2,4]$，$[0,1,0]$，$[1,3,2]$ 则不能形成算术级数。

------------

这题其实代码实现并不难，但是它的思维难度还是比较高的。许多同学在做的时候会想到 $\texttt{DP}$、$\texttt{DFS}$ 等算法，其实是完全没必要的。

我们可以尝试将三个数表示出来，并通过题目信息逐步确定它们的特点，最后利用等差数列的特点，通过另外两个数算出某一个数应该变成的值。

现在，我们设这个数列是个单调上升序列（右边比左边大的等差数列），并设第一个数为 $x$，公差为 $y$。那么：第二个数为 $x + y$，第三个数为 $x + 2y$。

反之，如果这个数列是单调下降序列，那么第二个数为 $x - y$，第三个数为 $x - 2y$。

先判断第一个数。通过等差数列的特点，我们用第二个和第三个数求这个数。

单调上升序列，因为 $2(x + y) - (x + 2y) = x$，所以第一个数等于两倍第二个数减去第三个数，即 $a = 2b - c$。

单调下降序列，因为 $2(x - y) - (x - 2y) = x$，所以第一个数等于两倍第二个数减去第三个数，即 $a = 2b - c$。

再判断第二个数。众所周知，第二个数为第一个数和第三个数的平均数，所以无论是什么序列，都有 $b = \frac{a + c}{2}$。

再判断第三个数。通过等差数列的特点，我们用第一个和第二个数求这个数。

单调上升序列，因为 $2(x + y) - x = x + 2y$，所以第一个数等于两倍第二个数减去第三个数，即 $c = 2b - a$。

单调下降序列，因为 $2(x - y) - x = x - 2y$，所以第一个数等于两倍第二个数减去第三个数，即 $c = 2b - a$。

综上所述，我们可以发现：**无论是哪一种序列，推算公式都是一样的。**

通过题目，我们不难看出：三个数都是 **正整数**，所以如果经过计算，应该使数列变成等差数列的数小于 $0$，那么这个数不合法，需要判断下一个数的可能。且因为 $m$ 是整数，所以如果 $a$ 要变成 $(2b - c)$，就要乘整数倍。也就是说，$(2b - c)$ 可以整除 $a$。以此类推，所有数在变得使数列变成等差数列时，它变成的那个数都能整除原来的数。如果这个使数列变成等差数列的数无法整除原数，则这个数无法变成使数列变成等差数列的数，接着判断下一个数的合法性。

现在可以轻松通过力！

#include<bits/stdc++.h>
using namespace std;
int t; //测试数据组数
int main(){
	scanf("%d", &t);
	for(int i = 1;i <= t;i++){
		int a, b, c;
		scanf("%d%d%d", &a, &b, &c);
		if(((2 * b - c) > 0 && (2 * b - c) % a == 0) || (a + c) % (b * 2) == 0 || ((2 * b - a) > 0 && (2 * b - a) % c == 0)) //可以通过乘整数m变成算术级数的情况
			printf("YES\n");
		else //否则
			printf("NO\n");
	}
	return 0;
}
