这道题目的大概意思是一段视频每秒用 $a$ 点数，每秒会增加 $b$ 点数（这些点数在这一秒可以直接用），视频一共有 $c$ 秒，求提前要等多少秒后的点数才能够放完这个视频。

这题不难，由于数据范围不大($a$,$b$,$c$<=$1000$)，使用公式或模拟都可以过。另一篇题解用的是公式，我就挂一个模拟代码。模拟用不着`long long`，使用`int`足够了。

如果每秒增加的点数比每秒消耗的点数少（$a$<=$b$）就可以直接输出$0$。

如果不是，可以先定义一个`point`来代表点数，然后进行死循环，每次让`point`加 $b$，然后判断一下`point`加上看视频时增加的点数（$b\times c$）是否大于看视频需要的点数（$a\times c$），如果是就输出循环变量（代表秒数）然后结束，不是就继续循环。

```cpp
#include<iostream>
using namespace std;
int main(){
    int a,b,c,point=0;//a、b、c同题目，point为点数
    cin>>a>>b>>c;//输入
    if(b>=a){//如果每秒增加的点数比每秒消耗的点数多
        cout<<0;
		return 0;
    } 
    for(int second=1;;second++){
        point+=b;
        if(point+b*c>=a*c){//如果点数够了
            cout<<second;
            return 0;//由于只需要输出一次，输出完直接结束就可以了，结尾也没必要再写一遍
        }
    }
}
```
