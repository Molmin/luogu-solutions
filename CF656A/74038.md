愚人节题目，打表。

其实就是输出2的n次方，但是2的13次方被算错了(变成8092了)，导致后面全错。本数列为[OEIS数列A221180](http://oeis.org/A221180)，似乎是当年达芬奇计算2^n时的错误。

代码:
```
#include <iostream>
using namespace std;
long long arrAns[36] = { 1, 2, 4, 8, 16, 32, 64, 128, 256, 512, 1024, 2048, 4096, 8092, 16184, 32368, 64736, 129472, 258944, 517888, 1035776, 2071552, 4143104, 8286208, 16572416, 33144832, 66289664, 132579328, 265158656, 530317312, 1060634624, 2121269248, 4242538496, 8485076992, 16970153984, 33940307968 }; //打表用，OEIS数列A221180
int main(){
	int iIndex; //索引
	cin >> iIndex; //输入索引
	cout << arrAns[iIndex]; //输出
	return 0; //告辞
}
```
