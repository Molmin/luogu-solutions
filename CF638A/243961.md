这道题的大概意思就是说给路两边的房子编号（其实就是挂门牌号），奇数在一边，偶数在一边。告诉你Vasya要去的房子的编号，问有多远。

大佬们是对于奇数和偶数进行分类讨论。但是 ~~（我不喜欢）~~ 分类讨论，很烦，还容易错。既然这题只有两类：奇数和偶数，那么是不是可以考虑：
##### 把偶数变成奇数！

观察题目中的图可以知道：房子都是面对面的。也就是说：

1号与n号的距离相等；

3号与n-2号的距离相等；

5号与n-4号的距离相等；

n-1号与2号的距离相等……

发现什么规律了吗？

相对的两个门牌号的数加起来一定是n+1！

所以我们就可以推出：第i号（i是偶数）房子与第n+1-i号房子是相对的，也就是距离相等的！

所以偶数转奇数的代码就呼之欲出了。

```cpp
if(a%2==0)//偶数判断。
	{
	  a=n-a+1;//将偶数门牌号通过相对的方式转换为奇数门牌号。
	}
```

现在我们可以只看上方那一串房子了，最让人头疼的两排问题解决掉了。

通过观察可以得出，这可以抽象成一个公差为2的等差数列。

我们要做的事情就是要求出公差的个数，但由于Vasya的车离1号房子还有1个单位长度，所以再加1。

等差数列求项数公式：（末项-首项）/公差+1。

故公差个数为：（末项-首项）/公差。

即（a-1）/2+1。

于是完整代码就有了：

```
#include<bits/stdc++.h>
using namespace std;
int main()
{
  	ios::sync_with_stdio(false);//cin加速器
        cin.tie(0); //cin加速器
	int n,a;
	cin>>n>>a;
	if(a%2==0)
	{
	  a=n-a+1;
	}
	cout<<(a-1)/2+1;
	return 0;
}


```
