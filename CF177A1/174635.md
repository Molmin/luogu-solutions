# 题解008 CF177A1

这是一道比较水的二维数组处理问题，哪怕是新手也可以很轻松的做出来，下面讲讲做法和一些注意点。

## 01 本题思路

我们此处的讨论默认下标从 $(1,1)$ 开始

### 中间列上的元素长什么样？

假定边长为 $n$ ，由于题目保证 $n$ 为奇数，因此倘若我们直接除以二，由于 $C++ int$ 除法向下取整的问题，显然答案是不对的。

```
a[1] a[2] a[3] a[4] a[5]
a[mid]=a[3]=a[(5+1)/2]
```
正确答案应当是 $(n+1)/2$ 。

### 中间行上的元素长什么样？

由于推导与中间列完全一致，本处跳过，下标也是 $(n+1)/2$ 。

### 斜向上对角线上的元素长什么样？

```
a[1][1] a[1][2] a[1][3] a[1][4] a[1][5]
a[2][1] a[2][2] a[2][3] a[2][4] a[2][5]
a[3][1] a[3][2] a[3][3] a[3][4] a[3][5]
a[4][1] a[4][2] a[4][3] a[4][4] a[4][5]
a[5][1] a[5][2] a[5][3] a[5][4] a[5][5]
```

观察这个例矩阵，我们可以发现对于 $a[i][j]$ 如果满足 $i+j=n+1$ ，那这个元素显然属于斜向上对角线。

### 斜向下对角线上的元素长什么样？

继续观察那个矩阵，我们可以发现对于 $a[i][j]$ 如果满足 $i=j$ ，那这个元素显然属于斜向下对角线。

## 02 本体算法

### 模拟

我们只需要遍历整个矩阵，然后找到所有满足01中条件的点加入总和即可。

但是值得注意的是，由于中心元素同时满足上述四个条件，因此我们应当手动去重。

## 03 AC代码

```cpp
#include<algorithm>
#include<cstdio>
#include<iostream>
using namespace std;
int n,a[200][200],sum;
int main(){
    cin>>n;
    for(int i=1;i<=n;i++){
        for(int j=1;j<=n;j++){
            scanf("%d",&a[i][j]);
        }
    }
    for(int i=1;i<=n;i++){
        for(int j=1;j<=n;j++){
            if(i+j==n+1) sum+=a[i][j];
            if(i==j) sum+=a[i][j];
            if(i==(n+1)/2) sum+=a[i][j];
            if(j==(n+1)/2) sum+=a[i][j];
        }
    }//遍历矩阵，寻找满足条件的点
    sum-=(3*a[(n+1)/2][(n+1)/2]);//手动去重
    cout<<sum;
}
```

## 04 易错点

### 04.01 下标问题

在上述讨论中我们的下标都是 $(1,1)$ 开始的，如果你从 $(0,0)$ 读入，那关系可能得重新进行处理。
