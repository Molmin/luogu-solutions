## 算法解析
对你读入的一行字符串进行处理，也就是 **统计非空格非换行符的字符** 。当然 $1 ≤ |s| ≤ 5$ 想超时是不可能的~~（打了 1000+ 个 sleep(1) 的请绕道）~~，你可以直接读完一行，或读入字符，或边读边做，或……
## 程序
这里介绍两种代码（这里只贴核心代码，剩下的留给你们自己去脑补了 $\color{orange}\text{创建美好洛谷，从不抄题解开始}$）

1. **直接读完一行**
```cpp
    getline(cin,s);//一种奇妙的字符串读入函数，可以自己设定读到什么停止，默认回车
    for(int i=0;i<s.size();i++)
    {
    	if(s[i]!=' ')
        {
        	ans++;
        }
	}//统计非空格的字符（换行符默认getline是不会读进字符串的）
    cout<<ans; 
```
当然要注意，如果要读入多行，且你用 $getline$ 读入其中字符串的话你需要 **读入一个字符** 或 **再 $getline$ 一个字符串** ，把换行符读掉，不然即使你在 $Windows$ 下测试没问题，实际上你读的字符串的首个字符在~~万恶的~~ $NOI$ $Linux$ 评测系统下竟是 \$n$ ~~坑爹~~换行符，这主要是因为 $NOI$ $Linux$ 评测系统的换行符有两个，分别是 #$10$ 和 #$13$，而 $Windows$ 只有 #$10$ 这一个。
