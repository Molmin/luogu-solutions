## 玄学做法（不知道为什么是对的）

我们建一棵树，根节点标为 * ，左子树是0,右子树是1。

根据题面写出第三层 0 1 1 0 0 1 1 0

那么我们发现从上向下读就是第n层的格雷码

如图：


![贴一张丑陋的图片](https://cdn.luogu.com.cn/upload/image_hosting/5figztgn.png?x-oss-process=image/resize,m_lfit,h_170,w_225)

可以比较清楚地看出最下边数的规律就是01100110……

这样我们从上到下依次输出就是所求的格雷码

例如n=3时，k=0则为000，，k=2则为011

这样已知k，通过不断的递推就可以得到格雷码。

核心代码贴上：

```cpp
while(n>0)
{
    if(k % 4 == 1||k % 4 == 2) geleima[n]==1;//k每4个数循环
    else geleima[n]==0;
    k=k/2;
    n--;
}
```

这里我们倒序存储正序输出即可。

### update: 这里给一下正确原因。

首先根据格雷码的性质，0000…… 一定是第0个格雷码。

那么根据字典序最小原则，一定是从最末一位开始逐渐变为1 。

而格雷码每次只能改变一位则确定了00000……11一定排在00000……10前

如图所示的树逐层向上的变化规律符合格雷码的性质。

所以
