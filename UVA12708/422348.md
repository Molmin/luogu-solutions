### 题目大意
本题乍一看，要死了，全是英文，赶紧拍照翻译。
这边帮大家解释一下：
大体意思就是给你一个数字（$n$），让你求从 $1$ 到 $n$ 任取两个数字求 $gcd$ (最大公约数)。 如果有不知道最大公约数的这边放了[链接](https://baike.baidu.com/item/%E6%9C%80%E5%A4%A7%E5%85%AC%E7%BA%A6%E6%95%B0/869308?fr=aladdin)。然后请你输出最大的那个最大公约数

### 分析
这题按照暴力查询,算法肯定要 $O(n^2)$ ，就算你的 $gcd$ 函数写的好到天边，暴力查询优化得很好，至少也要 $O(n)$ 。一看要 $TLE$ 了，本蒟蒻第一想到的就是打表来找找规律。打表不要全打出来，先打 $30$ 看看。打出来得结果是这样的：
```latex
1:0 2:1 3:1 4:2 5:2 6:3 7:3 8:4 9:4 10:5 11:5 12:6 13:6 14:7 15:7 16:8 17:8 18:9 19:9 20:10 21:10 22:11 23:11 24:12 25:12 26:13 27:13 28:14 29:14 30:15
```
稍微细心点就能发现，其实每个数所对应的最大公约数其实就是这个数除 $2$ 向下取整，然后整道题就变得比入门题还简单。

### 注意
 看好本题的数据范围，不开 $long long$ 可能会~~死的~~很惨。

### 代码
```cpp
#include<bits/stdc++.h>
using namespace std;
int main(){
	int a;
	long long n;
	while(a--){
		cin>>n;
		cout<<n/2<<endl;
	}
	return 0;
}
```
