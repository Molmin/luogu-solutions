# 题目分析
[题目传送门](https://www.luogu.com.cn/problem/CF689A)

迈克的手机掉水里了，他只能记住他在输入一些数字时的移动方式~~这是神马记性~~，所以他现在担心，有没有其他数字，有相同的手指动作？

首先要明白，这道题要解决的是**有没有其他数字，有相同的手指动作**，比如**568**和**253**的手指动作


![](https://cdn.luogu.com.cn/upload/vjudge_pic/CF689A/1ee471f119d0342dde68609c8abcbc49afcbf489.png)![](https://cdn.luogu.com.cn/upload/vjudge_pic/CF689A/030b9a0e4d0e91a9966bf07663da14ee536572b7.png)

有没有发现什么特点 

没错，只要三个数字都能向同一方向移动，那么就有相同的手指动作~~这不是废话吗~~，所以我们只需要判断是不是所有数字都可以向同一方向移动就可以了

# 坑
- 这个题的一个坑点 ---- 输入

来看第一个样例
```cpp
3
586

```

有木有注意到**下面的移动方式之间是没有空格的**

所以移动方式就不能直接用int读入

那我们可以这样

```

	char c;
	int m;

	cin>>c;
    	m = c - '0';
```

给输入来一个强制转换套餐岂不美哉


#### 接下来就是你们期待许久的

# 代码
```cpp
#include<iostream>
#include<cstdio>

using namespace std;

int n , m;
char c;
bool l , r , u , d;//上下左右四个方向

int main(){

    cin>>n;

    for( int i = 1; i <= n; i++ ){
        cin>>c;//输入单个字符
        m = c - '0';//类型转换(少打几个单引号他不香嘛)
        if( m == 6 || m == 3 || m == 9 || m == 0 ) r = 1;//判断四个方向可不可以走
        if( m == 1 || m == 2 || m == 3 ) u = 1;
        if( m == 1 || m == 4 || m == 7 || m == 0 ) l = 1;
        if( m == 7 || m == 9 || m == 0 ) d = 1; 
    }

    if( r == 0 || l == 0 || u == 0 || d == 0 ) cout<<"NO";//只要有一个方向可以走就有重复
    else cout<<"YES";

	return 0;//撒花`(°▽°)ノ
}
```
