# 水题我还差点错了，为了报复，我特意写了一篇题解：
## 我就差点错在那个码的长度上，本来以为它没用的，结果它起到了关键作用
## 所以大家以后做题一定要注意那些你觉得没用的东西
### 废话了这么多，上代码吧：
```cpp
#include<bits/stdc++.h>
using namespace std;
int a[1001],b[1001],s[1001],n,q;
int main(){
    cin>>n>>q;
    for(int i=0;i<n;i++)
		cin>>a[i];
	for(int i=0;i<q;i++)
	{
		cin>>s[i]>>b[i];
		int w=1;//w一定要设成1，不然乘了半天还是0
		for(int j=0;j<s[i];j++)//s[i]的关键作用，能让你省去很多步骤
			w*=10;
		int x=0x7f7f7f7f;//因为要求最小，所以把x设大
		for(int j=0;j<n;j++)
			if(a[j]%w==b[i])
			{
				if(x>a[j])//判断最小，注意：某些童鞋觉得>和>=差不多，但是，很多时候>=会交换更多次导致慢，有些时候，这两个简直是天差地别
					x=a[j];
			}
		if(x==0x7f7f7f7f)//如果x==原值就输出-1
			cout<<"-1"<<endl;
		else
			cout<<x<<endl;
	}
	return 0;
}
```
## 这个代码真会AC，因为洛谷测评机是把输入放一块测，输出放一块测。这是一个~~老师教的~~小技巧。
