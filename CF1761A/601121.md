对于不太能理解题的萌新，我给出一段我自己概括的题目描述。
## 题目概括
给出 $t$ 组数据，每组数据给出三个正整数数 $n$，$a$，$b$。请问是否存在两个长度为 $n$ 且内部由1到 $n$ 这 $n$ 个数排列而成的序列。使得这两个序列的前缀长度为 $a$ 的前缀序列相等，且长度为 $b$ 的后缀序列也相等。（前缀和后缀的意思去网上查）
## 做题思路
只有三个数，我们就要去考虑这三个数的关系。具体可以分为以下三个部分：

- $n = a = b$，结果：成立
- $a+b<n-1$，结果：成立
- $n-1\le a+b$ 并且不满足第一类条件，结果：不成立

第一类很好解释，通过样例中的第一组数据就可以知道前缀长度和后缀长度如果等于数组长度那么就相当于全部了，所以不管怎么排列都是成立的。

第三类可以这么想，如果成立，一定有多种排列，但如果在序列中，前缀与后缀相交，就已经确定了数组，必然不成立，如图。
![](https://img-blog.csdnimg.cn/65a47f45022a4b80ad3f52e1a00ed0ab.png)
但如果前缀和后缀相差一个位置，如图。由于数组中的数时1到 $n$ 并且不能重复，所以已经有 $n-1$ 个数被选走了，那最后一个空也就只有一种选择了，随意最终也无法组成多个序列，故也不成立。
![](https://img-blog.csdnimg.cn/9950f8ae0a2c45cd87931c8efd68aad3.png)
反之，第二类前缀与后缀相差的格数大于1。所以必然成立，例如：

$n=4$，$a=1$，$b=1$

那么就变成如图所示的样子，中间空着两个格子，而这两个格子就有两种排法，所以成立。
![](https://img-blog.csdnimg.cn/19cc4a68ee5241888cb323a432777164.png)
最终就可以得到以下代码。
## 通过代码
```cpp
#include<bits/stdc++.h>
using namespace std;
int t,n,a,b;//分别代表数据组数，以及题目中的n,a,b 
int main()
{
	cin>>t;
	while(t-- and cin>>n>>a>>b)//其实就是循环t次，每次输入n,a,b 
	{
		if(n==a and a==b or a+b+1<n) cout<<"Yes"<<endl;//半段如果满足条件1或者条件2就成立。 
		else cout<<"No"<<endl;	//否则一定是条件3，故不成立。 
	}
	return 0;
}
```
