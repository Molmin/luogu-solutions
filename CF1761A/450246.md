## 题意

[题目传送门](https://www.luogu.com.cn/problem/CF1761A)

给出 $3$ 个正整数 $n,a,b$，确定是否存在两个长度为 $n$ 的这样的排列 $p,q$：

- $p,q$ 的最长公共前缀长度为 $a$；
- $p,q$ 的最长公共后缀长度为 $b$。

排列的定义参考题面。

##### 注：如果你在原题中看到了这个翻译，说明我的翻译过审了。

## 分析

根据 $1\;1\;1$ 这组样例，我们可以得到：当 $a=b=n$ 时，存在满足条件的 $p,q$。显然，这时 $p=q$。所以此时为 `Yes`。

再看 $2\;1\;2;3\;1\;1$ 这两组输出为 `No` 的样例，我们发现：  
对于 $2\;1\;2$，如果要让 $p,q$ 最长公共后缀长度为 $2$，那么后缀也就变成了前缀（因为长度就是 $2$），所以应有 $a=b$，才能保证存在 $p,q$。但是 $a \ne b$，所以此时输出 `No`。

而对于 $3\;1\;1$，如果满足条件，那么 $p_1=q_1,p_3=q_3$。发现 $p_2,q_2$ 也只有一种选择了，即 $p_2=q_2$，所以不存在 $p,q$，输出 `No`。  

可以扩展为：**如果最长公共前缀和最长公共后缀有交集，或者它们之间的间距小于等于 $1$ 时，不存在满足条件的 $p,q$。**  
转换为式子就是：**当 $a+b\le n-1$ 时，不存在满足条件的 $p,q$。**

最后看样例 $4\;1\;1$。这时前缀、后缀之间的间距为 $2$，而有了这 $2$ 个位置就可以实现互换，从而阻挡最长公共前缀、最长公共后缀长度的扩展（假设我们现在在求它们的长度），所以存在满足条件的 $p,q$。

转换为式子就是：**当 $a+b<n-1$ 时，存在满足条件的 $p,q$。**

## 代码

代码很容易写出来：

```cpp
#include <bits/stdc++.h>
using namespace std;
#define No cout<<"No\n"
#define Yes cout<<"Yes\n"
int t,n,a,b;
int main()
{
    cin >> t;
    while(t--) // 恰好循环 t 次
    {
        cin >> n >> a >> b;
        if(a==b && b==n)
            Yes;
        else if(a+b>=n-1)
            No;
        else if(a+b<n-1)
            Yes;
    }
    return 0;
}
```

对于初学者：$\mathtt{\#define\;A\;B}$ 等价于将源码中所有的 $\mathtt{A}$ 替换为 $\mathtt{B}$。所以 `Yes;` 等价于 `cout<<"Yes\n";`。
