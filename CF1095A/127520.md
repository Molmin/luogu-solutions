#### 这道题得扯到一个小学概念：高斯求和（听起来NB实际上SOEASY）


------------


### 概念部分（知道高斯求和的跳过）


如果我们要计算1到100的和也就是1+2+3+......+99+100

我们可以将1和100看做一组，2和99看做一组，由于这是等差的，每一组的和都一定

需要求总和那就用每组的大小101乘组数100/2也就是5050

推广之后，求1+2+......+n，就用（n+1）乘n/2就可以了，简化后为n(n+1)/2



------------

### 知道了这个，题目就容易了（本蒟蒻tcl，时间复杂度更高，但无所谓，数据不大）

思路：先扫一遍得出原串长度，再通过高斯求和公式输出，具体在代码里说：

```
#include <bits/stdc++.h>   //万能头
using namespace std;
int n,i; 
string s;           //习惯用string
int main(){
cin>>n;
cin>>s;         //输入
for(i=1;i<=n;i++){        //循环枚举原串长度
if(i*(i+1)==2*n){
n=i;
break;      //如果这个原串长度是符合的就用n记录下来
}
}
for(i=1;i<=n;i++)
cout<<s[i*(i+1)/2-1];  //输出原串位置对应的字符即可
return 0;        //好习惯
}
```
